## 🧭 Research Direction

I am a backend engineer in Korea, currently exploring how **XAI, uncertainty, and agent traces** can be evaluated in practical systems.

My current focus is not to look like a finished researcher, but to turn engineering experience into small, testable research questions:

- **Medical XAI**: How can we evaluate whether an explanation map is faithful, not only visually plausible?
- **Uncertainty & Calibration**: Can uncertainty signals make XAI outputs more reliable in medical imaging or decision-support settings?
- **Agent Explainability**: What kind of agent trace actually helps humans debug, trust, or reject an AI agent's decision?
- **AI-assisted Software Engineering**: How should we evaluate code generated by AI agents beyond simple pass/fail tests?

> ICML 2026 preparation theme: **XAI × Medical Imaging × Agent Workflow Evaluation**

- **Agentic Workflow**: Break down product or engineering goals into planning, implementation, review, and QA loops
- **Prompt Engineering**: Design structured prompts for coding agents, reviewers, and research assistants
- **Validation Mindset**: Prefer reproducible checks, test cases, and explicit failure analysis over vague AI-generated outputs
- **Research Bridge**: Interested in making agent behavior more explainable and evaluable for real users

```text
1. Explanation Faithfulness

2. XAI + Uncertainty
   Can calibration or uncertainty estimation help users decide when an explanation should not be trusted?

3. Agent Trace Evaluation
   Which intermediate steps of an AI agent are useful for debugging, accountability, and human trust?

4. Practical Benchmarking
   Can real engineering tasks become a small benchmark for evaluating AI coding agents?
```
